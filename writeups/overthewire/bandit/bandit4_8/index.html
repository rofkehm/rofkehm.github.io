Metodología y Solución

Nivel 4 → 5

Nos logueamos con la contraseña del nivel anterior: 

voodoo@voodoo-pc $ ssh bandit4@bandit.labs.overthewire.org -p 2220
bandit4@bandit.labs.overthewire.org's password: 2WmrDFRmJIq3IPxneAaMGhap0pFhF3NJ

Listamos el contenido de la carpeta inhere/: 

bandit4@bandit:~/inhere$ ls
-file00  -file01  -file02  -file03  -file04  -file05  -file06  -file07  -file08  -file09

El reto nos dice: "The password for the next level is stored in the only human-readable file in the inhere directory."

Sabemos que hay un archivo legible por humanos entre todos estos. Usamos find para buscar archivos tipo file y filtramos con file --mime-type para encontrar el que sea text/*.

Construimos un mini-script inline con -exec: 

bandit4@bandit:~/inhere$ find . -type f -exec sh -c 'file --mime-type "$1" | grep -q text/ && cat "$1"' _ {} \;
4oQYVPkxZOOEOO5pTW81FB8j8lxXGUQw

Explicación paso a paso:
• find . -type f → busca todos los archivos en el directorio actual
• -exec sh -c '...' _ {} \; → por cada archivo encontrado, ejecuta un script
• file --mime-type "$1" → obtiene el tipo MIME del archivo
• grep -q text/ → verifica silenciosamente si es texto
• && cat "$1" → si es texto, lo imprime

Contraseña nivel 5: 4oQYVPkxZOOEOO5pTW81FB8j8lxXGUQw

Nivel 5 → 6

Nos conectamos: 

voodoo@voodoo-pc $ ssh bandit5@bandit.labs.overthewire.org -p 2220
bandit5@bandit.labs.overthewire.org's password: 4oQYVPkxZOOEOO5pTW81FB8j8lxXGUQw

Vemos muchas carpetas: 

bandit5@bandit:~/inhere$ ls
maybehere00  maybehere01  maybehere02  ...  maybehere19

El reto: archivo con 1033 bytes, human-readable, no ejecutable.

Usamos find con filtro de tamaño: 

bandit5@bandit:~/inhere$ find . -type f -size 1033c | xargs cat
HWasnPhtq9AVKe0dmk45nxy20cvUa6EG

-size 1033c → busca archivos de exactamente 1033 bytes
xargs cat → imprime el contenido del archivo encontrado

Contraseña nivel 6: HWasnPhtq9AVKe0dmk45nxy20cvUa6EG

Nivel 6 → 7

Conexión: 

voodoo@voodoo-pc $ ssh bandit6@bandit.labs.overthewire.org -p 2220
bandit6@bandit.labs.overthewire.org's password: HWasnPhtq9AVKe0dmk45nxy20cvUa6EG

El archivo está en todo el servidor con:

  • Dueño: bandit7
  • Grupo: bandit6
  • Tamaño: 33 bytes

No hay nada en el directorio actual. Buscamos desde la raíz: 

bandit6@bandit:~$ find / -user bandit7 -group bandit6 -size 33c 2>/dev/null | xargs cat
morbNTDkSW6jIlUc0ymOdMaLnOlFVAaj

find / → busca en todo el sistema
2>/dev/null → ignora errores de permisos

Contraseña nivel 7: morbNTDkSW6jIlUc0ymOdMaLnOlFVAaj

Nivel 7 → 8

Conexión: 

voodoo@voodoo-pc $ ssh bandit7@bandit.labs.overthewire.org -p 2220
bandit7@bandit.labs.overthewire.org's password: morbNTDkSW6jIlUc0ymOdMaLnOlFVAaj

Tenemos un archivo grande: 

bandit7@bandit:~$ ls
data.txt

El reto: contraseña al lado de la palabra millionth.

Usamos grep: 

bandit7@bandit:~$ cat data.txt | grep millionth
millionth	dfwvzFQi4mU0wfNbFOe9RoWskMLg7eEc

grep millionth → busca líneas que contengan esa palabra

Contraseña nivel 8: dfwvzFQi4mU0wfNbFOe9RoWskMLg7eEc

Comandos Clave Aprendidos

  • find . -type f – Buscar archivos
  • file --mime-type – Identificar tipo de archivo
  • -size 1033c – Filtrar por tamaño exacto
  • -user / -group – Filtrar por propietario
  • grep "texto" – Buscar en archivos
  • 2>/dev/null – Ocultar errores
